📌 概述

本手册面向使用 Claude Code（命令行 AI 编程助手）的用户。meetings 技能把会议转录文本自动转成正式会议纪要 Word 文档，核心三步：知识库纠错 → 信息提取 + 口语书面化 → 模板填充。

💡
Claude Code 是命令行工具，功能全、可自定义模型、处理较快；但需装 CLI、配登录，入门门槛略高。若你更想要「下客户端、扫码即用」的简单体验，请看 Workbuddy 版手册。

🛠️ 安装 Claude Code

1

安装 Claude Code CLI

在 PowerShell 中运行：

irm https://claude.ai/install.ps1 | iex
2

首次登录

claude

按提示完成 Anthropic 账号登录或 API Key 配置。如需用国产模型，见「常见问题」。

📖
完整文档：Claude Code 官方文档

🧩 安装技能

meetings（出纪要）和 iTingnao（音频转写）都是技能。装好 meetings 即可用；要处理音频还需装 iTingnao。

📁 方法一：手动放置（推荐）

把技能目录放到：

$env:USERPROFILE\.claude\skills\meetings\

目录里至少要有 SKILL.md。config.json 在首次运行时自动生成，无需手动放入。

🧩 方法二：插件市场

/plugin /plugin marketplace add owner/repo

在市场浏览安装，插件内技能自动可用。

🐙 方法三：Git 克隆

git clone <仓库> "$env:USERPROFILE\.claude\skills\meetings"

便于后续 git pull 更新。

⚠️
处理音频文件需额外装 iTingnao 技能并配 API Key（/itingnao config）。只有文本文件则不用。

⚙️ 配置（首次自动完成）

无需手动编辑。首次运行 /meetings 时，技能自动选定工作目录、创建子目录并生成 config.json。

✅
业务人员：直接运行 /meetings，首次确认工作目录（默认 Documents\ai-meeting）即可，其余自动完成。
1

首次运行自动生成 config.json

技能检测到没有 config.json 时会自动创建（内容如下），project_root 即工作目录：

{ "project_root": "C:\\Users\\你的用户名\\Documents\\ai-meeting" }
⚠️
其余配置（registry.json / profile.json / profiles/ 目录）关系到模板与脚本匹配，不要轻易改动。知识库三件套的维护见部署说明。

📝 使用 meetings

按你的起点选择：已有文本稿 → 起点 1；只有录音 → 起点 2。

📌 起点 1：文本文件（不用 iTingnao）

已有转写好的文本稿，直接交给 meetings：

/meetings C:\会议\4月例会.txt 这是例会，按例会出纪要

给路径 + 一句自然语言（如「这是例会」），大模型据此识别会议类型，不用 --type。

📌 起点 2：音频文件（一条指令调两次技能）

只有录音、还没转写时，一条指令里先后调用 iTingnao（转写）+ meetings（出纪要）：

/itingnao transcribe C:\会议\4月例会.mp3 /meetings <转写文本> 这是例会，按例会出纪要

也可在一条需求里直接说「先转写这段录音再按例会出纪要」，让大模型自动串起两次技能调用。

🔧 需要更精确时：加参数

/meetings C:\会议\4月例会.txt \ --type 例会 \ --format 纪要 \ --mode 汇报会
参数说明
--type指定会议类型配置套（最关键的定位参数）
--format输出格式：督办单 | 纪要
--mode会议模式：汇报会 | 讨论会 | 评审会
--existing合并已填写的 Word 文件
💡
多数会议不用带参数——一句自然语言或转录里的关键词通常足够。只有关键词不明显、或想要非默认输出时才加参数。

✅ 核对归档

1

查看输出文件

① 会议纪要 Word 文档 ② 来源标注（每条信息的来源时间戳与原文，可追溯）。

2

核对「待确认」字段

纪要中标注「[待确认]」的字段（如日期推断、疑似责任人）需人工核实后定稿。

❓ 常见问题

/meetings 提示 Unknown skill？

检查 ~/.claude/skills/meetings/SKILL.md 是否存在，重启 Claude Code 后再试。

能用国产模型吗？

可以，通过 API 代理接入兼容 OpenAI 格式的国产模型（DeepSeek / 通义 / 智谱等）。复杂评审会建议优先用 Claude Sonnet/Opus。

说话人识别不准？

Step 1.5 会确认参会人员，仅在该名单内匹配说话人。
