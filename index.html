📌 这是什么

meetings 是一个 AI 技能：输入带时间戳与说话人的会议转录文本，输出一份可直接归档的正式会议纪要 Word 文档。它跑在大模型上，可在 Claude Code（命令行）或 Workbuddy（腾讯客户端）里使用。

💡
所谓"技能"就是一份给大模型的工作说明书（SKILL.md）加一套知识库。装好后用一句 /meetings <转录文本> 这是例会 就能触发整个流程，无需写代码。

📥 输入

  • 会议转录文本（含时间戳 + 说话人）
  • 一句自然语言说明（如"这是例会"）
  • 可选：已填写的 Word 纪要（合并复用）

📤 输出

  • 正式会议纪要 Word 文档
  • 来源标注文件：每条信息可追溯到原文时间戳与说话人

🎯 能做

  • ASR 纠错、说话人识别、责任归属核验
  • 口语 → 公文书面化转写
  • 按会议类型套用模板（例会 / 评审 / 汇报…）

⚙️ 工作流程

核心四步：读取与匹配 → 知识库纠错 → 信息提取与书面化 → 模板填充。

1

读取转录文本 + 配置套匹配

按会议类型（例会 / 评审 / 汇报…）匹配对应的模板与知识库配置套；可选读取已填写的 Word 纪要合并复用。

2

知识库纠错

ASR 同音错 / 拆分错纠正、地名与系统名别称替换、业务术语规范；同时识别说话人、核验责任归属。

3

信息提取 + 口语书面化

提取会议名称、时间、参会人、工作安排等字段，并把口语转写为公文书面语，确保每条内容都能对应原文。

4

模板填充 + 来源标注

把结构化结果填入 Word 模板生成纪要，并生成来源标注文件——每条工作安排都能追溯到时间戳、说话人与原文。

📚 知识库三件套

技能依赖三个 Markdown 文件（放在 project_root/profiles/ 下），是用户唯一需要维护的内容——改人名、改术语、改输出规则都只动这里。

关键字纠错.md

ASR 纠错词条、地名 / 系统名别称、业务术语规范与释义。

公司概况.md

公司信息、人员档案、部门职责、地名 / 系统详情。

用户需求.md

输出格式、会议模式、模板映射、书面化规则、内容要求。

✅
新增人员 / 部门 / 纠错词条，只需在对应表格里加一行，无需改动技能本身。
